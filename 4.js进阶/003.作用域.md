# JavaScript 作用域详解

## 一、什么是作用域

作用域是指在程序中定义变量的可访问范围，决定了变量在何处可以被访问。JavaScript 中的作用域控制着变量的可见性和生命周期。

## 二、作用域类型

### 1. 全局作用域 (Global Scope)

在代码中任何地方都能访问到的变量，拥有全局作用域。

```javascript
var globalVar = '我是全局变量';

function test() {
  console.log(globalVar); // 可以访问
}
```

### 2. 函数作用域 (Function Scope)

在函数内声明的变量只能在函数内部访问，外部无法访问。

```javascript
function myFunction() {
  var localVar = '我是局部变量';
  console.log(localVar); // 可以访问
}

myFunction();
console.log(localVar); // ReferenceError: localVar is not defined
```

### 3. 块级作用域 (Block Scope)

ES6 引入的 `let` 和 `const` 声明的变量具有块级作用域，仅在声明它们的块 `{}` 内部可访问。

```javascript
if (true) {
  let blockVar = '我是块级变量';
  const blockConst = '我也是块级常量';
  var funcVar = '我是函数作用域变量';
}

console.log(funcVar);    // '我是函数作用域变量'
console.log(blockVar);   // ReferenceError: blockVar is not defined
console.log(blockConst); // ReferenceError: blockConst is not defined
```

## 三、词法作用域 (Lexical Scope)

JavaScript 使用词法作用域（静态作用域），即作用域由代码定义变量的位置决定，而非代码执行的位置决定。

```javascript
const value = 'global';

function outer() {
  const value = 'outer';
  
  function inner() {
    console.log(value); // 'outer'，查找最近的外部作用域
  }
  
  inner();
}
```

## 四、作用域链 (Scope Chain)

当访问一个变量时，JavaScript 引擎会先在当前作用域查找，若找不到则向外层作用域查找，直到找到该变量或到达全局作用域。

```javascript
const global = '全局';

function outer() {
  const outerVar = '外部';
  
  function inner() {
    const innerVar = '内部';
    console.log(innerVar); // '内部'
    console.log(outerVar); // '外部'
    console.log(global);   // '全局'
  }
  
  inner();
}
```

## 五、变量声明与作用域

### 1. var 声明

- 函数作用域
- 存在变量提升
- 可以重复声明
- 在全局作用域声明会成为 window 对象的属性

### 2. let 声明

- 块级作用域
- 不存在变量提升（存在暂时性死区）
- 不能重复声明
- 全局声明不会成为 window 对象的属性

### 3. const 声明

- 与 let 相同的作用域规则
- 必须初始化值
- 不能重新赋值（但对象属性可以修改）

## 六、变量提升 (Hoisting)

JavaScript 引擎在执行代码之前会将变量和函数声明"提升"到作用域顶部。

### 变量提升

```javascript
console.log(name); // undefined (不是 ReferenceError)
var name = '张三';

// 等价于：
var name;
console.log(name);
name = '张三';
```

### 函数提升

```javascript
sayHello(); // "你好"
function sayHello() {
  console.log("你好");
}

// 函数表达式不会被提升
sayHi(); // TypeError: sayHi is not a function
var sayHi = function() {
  console.log("Hi");
};
```

## 七、暂时性死区 (Temporal Dead Zone)

`let` 和 `const` 声明的变量从代码块开始到声明语句之前的区域称为"暂时性死区"，在此区域访问变量会抛出错误。

```javascript
{
  // TDZ 开始
  console.log(value); // ReferenceError
  let value = 123;    // TDZ 结束
}
```

## 八、闭包 (Closure)

闭包是函数及其词法环境的组合，允许函数访问并操作函数外部的变量。

```javascript
function createCounter() {
  let count = 0;
  
  return function() {
    count++;
    return count;
  };
}

const counter = createCounter();
console.log(counter()); // 1
console.log(counter()); // 2
```

## 九、模块作用域

ES6 模块拥有自己的作用域，通过 `export` 和 `import` 控制变量的可访问性。

```javascript
// module.js
const privateVar = '私有变量';
export const publicVar = '公共变量';

// main.js
import { publicVar } from './module.js';
console.log(publicVar);    // '公共变量'
console.log(privateVar);   // ReferenceError
```

## 十、执行上下文 (Execution Context)

与作用域密切相关的概念，分为全局执行上下文、函数执行上下文和 eval 执行上下文。每个上下文包含变量对象、作用域链和 this 值。

# JavaScript 作用域面试题

## 1. 变量提升与暂时性死区

```javascript
console.log(a);
console.log(b);
console.log(c);

var a = 1;
let b = 2;
const c = 3;
```

**答案**：
- `a`: `undefined`（var 声明提升，但赋值不提升）
- `b`: `ReferenceError`（let 存在暂时性死区）
- `c`: 代码不会执行到这里，因为前一行已经抛出错误

## 2. 闭包与循环变量

```javascript
const funcs = [];
for (var i = 0; i < 3; i++) {
  funcs[i] = function() {
    console.log(i);
  };
}

funcs[0]();
funcs[1]();
funcs[2]();
```

**答案**：全部输出 `3`

**解释**：由于使用 `var` 声明，`i` 是函数作用域变量，循环结束后 `i` 值为 3。所有函数共享同一个 `i` 引用。

**修复方法**：
```javascript
// 方法一：使用 let
for (let i = 0; i < 3; i++) {
  funcs[i] = function() {
    console.log(i);
  };
}

// 方法二：使用 IIFE 创建闭包
for (var i = 0; i < 3; i++) {
  funcs[i] = (function(value) {
    return function() {
      console.log(value);
    };
  })(i);
}
```

## 3. 闭包实现私有变量

```javascript
function createStack() {
  const items = []; // 私有变量
  
  return {
    push(item) {
      items.push(item);
    },
    pop() {
      return items.pop();
    },
    getItems() {
      return items;
    }
  };
}

const stack = createStack();
```

**问题**：上述代码中 `getItems` 方法暴露了私有变量，如何改进以保持私有性？

**答案**：
```javascript
function createStack() {
  const items = []; // 私有变量
  
  return {
    push(item) {
      items.push(item);
    },
    pop() {
      return items.pop();
    },
    size() {
      return items.length;
    }
    // 移除 getItems 方法
  };
}
```

## 4. 函数声明与函数表达式的提升

```javascript
function foo() {
  function bar() {
    return 3;
  }
  
  return bar();
  
  function bar() {
    return 8;
  }
}

console.log(foo());
```

**答案**：`8`

**解释**：函数声明会被提升到当前作用域顶部，后面的 `bar` 函数覆盖了前面的声明。

## 5. 块级函数声明（ES6+）

```javascript
'use strict';
if (true) {
  function foo() { return 1; }
} else {
  function foo() { return 2; }
}
console.log(foo());
```

**答案**：在严格模式下，会抛出 `ReferenceError`

**解释**：在严格模式下，块级函数声明只在声明它们的块内有效。

## 6. let 与循环

```javascript
for (let i = 0; i < 3; i++) {
  let i = 'abc';
  console.log(i);
}
```

**答案**：打印三次 `'abc'`

**解释**：循环内部的 `let i` 声明创建了一个新的变量，与循环变量 `i` 在不同的作用域。

## 7. var、let、const 在全局作用域的区别

```javascript
var a = 1;
let b = 2;
const c = 3;

console.log(window.a);
console.log(window.b);
console.log(window.c);
```

**答案**：
- `window.a`: `1`
- `window.b`: `undefined`
- `window.c`: `undefined`

**解释**：用 `var` 声明的全局变量会成为 window 对象的属性，而 `let` 和 `const` 声明的变量不会。

## 8. 执行上下文与作用域链

```javascript
var x = 10;

function foo() {
  console.log(x);
}

function bar() {
  var x = 20;
  foo();
}

bar();
```

**答案**：`10`

**解释**：JavaScript 使用词法作用域，`foo` 函数的作用域链在函数定义时确定，指向全局作用域的 `x`。

## 9. 闭包陷阱

```javascript
var data = [];

for (var i = 0; i < 3; i++) {
  data[i] = function() {
    console.log(i);
  };
}

data[0]();
data[1]();
data[2]();
```

**答案**：全部输出 `3`，与问题 2 类似。

**进阶问题**：如果需要实现输出 0, 1, 2，请给出三种解决方案。

**答案**：
```javascript
// 方案一：使用 let
for (let i = 0; i < 3; i++) {
  data[i] = function() {
    console.log(i);
  };
}

// 方案二：使用 IIFE
for (var i = 0; i < 3; i++) {
  data[i] = (function(index) {
    return function() {
      console.log(index);
    };
  })(i);
}

// 方案三：使用 bind
for (var i = 0; i < 3; i++) {
  data[i] = console.log.bind(null, i);
}
```

## 10. 实现记忆函数 (Memoization)

```javascript
function memoize(fn) {
  // 实现一个记忆函数，缓存计算结果
}

const factorial = memoize(function(n) {
  if (n <= 1) return 1;
  return n * factorial(n - 1);
});

console.log(factorial(5)); // 120，只计算一次
console.log(factorial(5)); // 120，直接使用缓存
```

**答案**：
```javascript
function memoize(fn) {
  const cache = {};
  
  return function(...args) {
    const key = JSON.stringify(args);
    if (cache[key] === undefined) {
      cache[key] = fn.apply(this, args);
    }
    return cache[key];
  };
}
```

## 11. 模块模式与闭包

```javascript
const calculator = (function() {
  let result = 0;
  
  function add(value) {
    result += value;
    return this;
  }
  
  function subtract(value) {
    result -= value;
    return this;
  }
  
  function getResult() {
    return result;
  }
  
  return {
    add,
    subtract,
    getResult
  };
})();

// 问题：如何实现链式调用？
```

**答案**：已经实现了链式调用，因为 add 和 subtract 方法返回了 this。示例：
```javascript
calculator.add(5).subtract(2).add(10).getResult(); // 13
```

## 12. 变量声明的优先级

```javascript
var x = 1;

function foo() {
  console.log(x);
  var x = 2;
}

foo();
```

**答案**：`undefined`

**解释**：函数内部的变量声明会提升到函数顶部，覆盖外部变量，但赋值不会提升。
